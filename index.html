Projects

Pixel Party

4-man project to showcase our ability to use React, CSS, Node.js, Express, and MongoDB Atlas.

Weeks 1-2

The goal was to making a functional multiplayer game where a player can draw an image on the screen, and the other members of the lobby guess the word in the chatbox. We spent sprint 1 setting up the login/register rest api calls.

Weeks 3-6

All aspects of the game including chat, drawing, and word-selection use the WebSocket package on the frontend and backend. This is essentially two singletons playing ping pong, both waiting for keywords to callback functions for the server/client to process.

Conclusion

Our biggest hurdles were getting the websockets to sync across devices. This included getting the drawing aspect to work, as well as running the whole show on a timer that could allow users to join the game at any time.

Raspberry Pi Camera Die Counter

This is a 3-man project where we were instructed to make a raspberry pi hardware setup which could use a python script to execute any hardware such as a breadboard with buttons. We chose to use a camera as an input device.

Design and Planning

In order to make a complicated python script, we used a AI-assistant to build a script that could not only take an image, but isolate the black dies on the image and return the correct sum. We needed a platform to build this on so I drafted a 3-d box-like structure.

Final Release

Initially the code would not recognize all the dice correctly. Using some prompt-driven development, I guided it to correct the errors in the image and successfully isolated the individual dies on screen.
