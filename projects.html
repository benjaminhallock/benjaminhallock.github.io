Pixel Party

4-man project to showcase our ability to use React, CSS, Node.js, Express, and MongoDB Atlas

Weeks 1-2

Sprint 1 was spent working out the details of the game and getting a working full-stack with authentication login.

Weeks 3-4

We started building the core game which required setting up a lobby with a list of players. Any player can join by copying the link to the lobby into the URL bar or Create Lobby page.

Weeks 5-6

Up until this point, were were sending REST API requets for all the data from front to back. For the game we needed to utilize websocket packages for Node/React. This requires creating a singleton on both the front and the back end, which actively monitors for any communication from a client/server, then callbacks to the appropriate functions. All aspects of the game including chat, drawing,

Week 5-6

The entire game runs on timers associated with the server. The server sends out new events to update the client. Once the "drawer" picks a word, they have a limited time to draw and let others guess the word in the chat box. Any correct guesses are show but not reveal. Once time runs out or everyone guessed, the round ends.

Weeks 7-8

The round restarts with a new drawer. The letters above the drawing can slowly be revealed at the game-leader's chosen speed. Once everyone has drawn once, we calculate the score based on who guess right the fastest, and who drew correctly enough to guess. A winner is decided, or the second round starts.

Conclusion

A functional game with user control and sound effects was created using a full-stack MERN application. We have working login with validation using REST API. We have some variables to control how the game operates. We handled multiple users logging in at different times to accomidate the gameplay. Live drawing can be done by all lobby users before the game starts. The game runs on a timer thanks to the WebSocket singleton on the Node server ochestrating when the clients change views.

Raspberry Pi Camera Die Counter

This is a 3-man project where we were instructed to make a raspberry pi hardware setup which could use a python script to execute any hardware such as a breadboard with buttons. We chose to use a camera as an input device. I wanted to read and image and output data on screen, so we chose dice as the calculation.

Design and Planning

In order to make a complicated python script, we used a AI-assistant to build a script that could not only take an image, but isolate the black dies on the image and return the correct sum. We needed a platform to build this on so I drafted a 3-d box-like structure using a web-app, then printed it in the schools 3d lab.

Initial Build

Initially the code would not recognize all the dice correctly. Using some prompt-driven development, I guided it to correct the errors in the image and successfully isolated the individual dies on screen.
